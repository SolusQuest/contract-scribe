# M0.4 framework-dependent Roslyn/MSBuild loading experiment

This record describes the public synthetic baseline for Issue #7. It is evidence for M0.5 and M0.6, not a production loader contract, audit-result contract, or distribution decision.

The fixture is `tests/fixtures/roslyn-msbuild/v1/Sample.sln`. It contains exactly two SDK-style C# projects: `SampleApp`, which has one project reference to `SampleLibrary`, and `SampleLibrary`. The expected graph and expected symbol set are handwritten in `expected-symbols.json`; the runner never generates its own oracle.

The semantic comparison artifact is the standalone `semantic-payload.json` file. It is written as UTF-8 without a BOM, whitespace, or trailing newline by a fixed `Utf8JsonWriter` property order. Projects and symbols use ordinal ordering. The comparison excludes environment, diagnostic, path, timestamp, duration, and process fields.

The required protocol is to restore once, run the framework-dependent host twice in fresh processes with `--no-restore`, compare the semantic payload bytes, and then run focused and full repository validation. Runtime execution performs no network or credential access. Ubuntu latest and Windows latest are required CI cells for closure.

The frozen M0.5 handoff consists of the fixture, expected oracle, semantic runner behavior, `(logicalProjectId, documentationCommentId)` identity, semantic payload, and ordering. M0.5 may change only host/deployment invocation and publish settings. A required frozen-artifact change is a negative M0.5 result and requires a new framework-dependent baseline.

The transfer manifest records the package/SDK baseline, commands, comparison contract, content hashes, known warnings, unresolved constraints, and public-safe evidence boundary.
