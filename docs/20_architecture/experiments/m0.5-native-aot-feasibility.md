# M0.5 Native AOT feasibility experiment

This record defines the public evidence boundary for Issue #8. It is an experiment input to M0.6, not a distribution decision or a general Native AOT support claim.

The experiment uses the frozen M0.4 fixture, expected-symbol oracle, framework-dependent semantic runner, identity and ordering rules, and canonical `semantic-payload.json` bytes. The Native AOT host is a separate test-only sibling project that calls the existing `FrameworkDependentExperiment` directly. It does not fork the runner, add AOT-specific semantic behavior, delegate to a framework-dependent child process, or read precomputed payloads.

The required matrix is Ubuntu latest with `linux-x64/X64` and Windows latest with `win-x64/X64`. The primary publish profile is `net10.0`, Release, `PublishAot=true`, `SelfContained=true`, `PublishTrimmed=true`, and the matrix-supplied runtime identifier. The normal solution restore and build remain RID-neutral; Native AOT restore, publish, and execution are separate steps.

Each cell runs the framework-dependent M0.4 protocol once, which itself launches two fresh host processes, then performs a clean RID-specific restore, clean Native AOT publish, and two direct fresh-process native executions. A positive cell requires both AOT payloads to be byte-identical to one another and to the framework payload. The native executable itself is never compared byte-for-byte.

The closed per-cell outcomes are `feasible-clean`, `feasible-with-warnings`, `not-feasible`, and `inconclusive`. Missing native prerequisites, incomplete restore, unknown classifications, M0.4 baseline failure, and non-reproducible observations are inconclusive. A reproducible unchanged-path incompatibility with complete evidence can be not-feasible. Protocol failures—missing, stale, malformed, contradictory, unsafe, or provenance-invalid artifacts—produce no cell outcome and exit non-zero.

The versioned evidence schema and registry are `schemas/experiments/m0.5-native-aot-evidence-v1.schema.json` and `docs/20_architecture/experiments/m0.5-native-aot-registry-v1.json`. The independent manifest binds the M0.4 transfer-manifest hash, frozen source revision, implementation inputs, publish profile, matrix, evidence paths, and invalidation rules. Evidence is canonical UTF-8 without BOM, insignificant whitespace, or trailing newline; commands are argument arrays, warnings and cells are ordinal-sorted, and only closed public toolchain fields are recorded.

The first closure is a one-time Ubuntu/Windows validation that produces sanitized per-cell records and an aggregate summary. Future pull requests run schema, registry, manifest/provenance, focused protocol, build, format, and CLI checks, but do not require mutable `*-latest` Native AOT toolchains. Toolchain or warning drift after closure requires a new evidence version and explicit revalidation issue.
