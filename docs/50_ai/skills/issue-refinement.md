# Issue refinement procedure

Before creating or refining an issue, read the relevant roadmap and architecture documents. State the problem, goal, bounded scope, exclusions, acceptance criteria, dependencies, validation, and public-safety constraints.

Use a separate issue when work requires an independent decision, experiment, pull request, or unblock state. For experimental work, define the question, fixture or evidence source, expected observation, and decision it can inform. Do not promise implementation outcomes that the experiment cannot establish.
